SQLAlchemy用法总结

发表于 2017-07-28 | 分类于 SQLAlchemy

可以用from SQLAlchemy import func 来用sql里面的函数，如 func.count('User.id') 或者 func.count('*')

在flask-sqlalchemy里使用SQL函数是用 db = SQLAlchemy() db.desc() db.count('*') 来使用

  • 条件查询
    • 还可以用 User.name.in_([‘qwe’, ‘asd’]) 来用in方法。 另外new_user = User(‘name’, ‘passwd’) 新建用户后， 执行db.commit() 后， new_user会自动有了uid，意思就是在commit之后， 那个instance 会刷新， 里面的属性会拥有数据库自动生成的属性 。 同in 还有 is, like, ilike(不区分大小写), ~User.name.in(not in), isnot(比如None, 和 != None 相同，判定不为空), or, and_, match
    • User.name.label(‘another_name’) 相当与SQL里的 select name asother_name form users.
    • 可以在query后面有多个filter， 相当与SQL里的 and.

  • 结果筛选

    • all() 把所有查询到的结果作为一个列表返回， 如果没有返回空列表

    • first() 只返回查询到的第一个结果， 如果无， 返回None

    • one() 如果查询结果是多个或者为空， 就raise一个异常。

    • one_or_none() 如果没有数据就返回None, 多个数据就raise异常。

ArchLinux--更新后无法挂载EFI分区

发表于 2017-07-28 | 分类于 Linux

=========更新===========

在我的home看到一个lib文件夹，用 sudo rm -rf lib/ 删除了，但是什么命令也都不能用了，并且开机提示没法找到主分区，GG!

换了manjaro linux kde桌面，不太好用挺慢的，不过manjaro linux挺不错的，是archlinux的衍生版，如果重新安装我会选择xfce桌面，先用着吧。


========分割线==========

在我的archlinux手贱不知怎么就升级了openssl后，各种依赖ssl的程序突然就不能用了，wiki是说:

pacman 完全坏掉，如何修复?

如果 pacman 完全坏掉不能使用，需要手动下载或构建需要的软件包>(openssl, libarchive 和 pacman) 并解压到根目录。pacman 会和默认配置文件一起恢复。之后，用 pacman 重新安装这些软件包以保证数据库的完整性。

放下面pacman会你妈的自动修复？，后来又去archlinux的bbs的pacman版块看，是说:

mirror修改,以及把各个包的名字都是最新的了，然后无效的-f选项，我以为是–force就可以，狗屁！

用最笨的方法了只能。

因为libcurl先报错，把curl升级最新，后来ssh2报错，升级ssh2，ssh2依赖dnssce,再下载它，一步一步更新，最后终于能pacman -Syy了，终于忍不住-Su了，又是各种文件已经存在不能更新，直接pacman -Syyu –force，更新！！！！

然后：

以前就有耳闻archlinux滚挂的传统，不负众望，开机就挂了，不能挂载EFI分区，我以为是fstab有问题，用的uuid来挂载EFI，把它换成partuuid，不行，直接/dev/sda1,不是不行，但是最初留了个心眼，bios里还有grub启动选项，试试grub启动，竟然可以进去！！！！！！

思考: 如果能进去没有出现挂载问题就肯定不是fstab的问题，而EFI分区启动步骤是用的efi文件读取efi分区里的archlinux启动配置，引导系统的是在EFI分区里的系统引导文件，而grub是用的/boot目录的引导文件，所以可能是更新系统后引导文件也更新了，过去的不能用了。

于是把两个文件cp进EFI分区，ok，可以了。
